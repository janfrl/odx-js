`@bc8-odx/proxy` provides the server-side layer for ODX, handling authentication, CSRF pre-fetching, and proxying requests to SAP backends.

## Features

-   **BTP Destination Support**: Integrated with `@sap-cloud-sdk/connectivity` to resolve destinations at runtime.
-   **Security Policies**: Fine-grained control over allowed methods, paths, and headers.
-   **CSRF Management**: Automatically fetches and rotates SAP CSRF tokens.
-   **Streaming Support**: Lightweight proxying with minimal memory overhead.

## Configuration

The proxy is typically configured via the Nuxt module, but it can also be used as a standalone Nitro module.

```ts [nitro.config.ts]
export default defineNitroConfig({
  modules: ['@bc8-odx/proxy/nitro'],
  odata: {
    services: [
      { name: 'MyService', url: '...', destination: 'MY_DESTINATION' }
    ]
  }
})
```

## BTP Destinations

When deployed to SAP BTP, ODX can leverage the Destination Service.

| Option | Description |
| --- | --- |
| `destination` | The name of the destination in BTP. |
| `proxyMode` | `stream` (default) or `buffer`. |

For a step-by-step guide on setting up destinations, see [BTP Destination Setup](/how-to/deployment).
