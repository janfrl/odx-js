# Quick Start

Once you have [installed and configured](/getting-started/installation) your service, you can start fetching data with full type safety.

## 1. Prepare Types

Run the Nuxt prepare command to generate the TypeScript models from your SAP EDMX schema.

```bash
npx nuxi prepare
```

## 2. Fetch Data

In your Vue component, use the `useOData` composable. It provides an intuitive dot-notation syntax for accessing your services and entity sets.

```vue [app.vue]
<script setup lang="ts">
// Autocomplete works for 'MyService' and its 'Products' entity set
const { data, pending, error } = await useOData().MyService.Products.list({
  $select: ['ID', 'Name', 'Price'],
  $filter: "Price gt 100.00"
})
</script>

<template>
  <div v-if="pending">Loading...</div>
  
  <div v-else-if="error">
    Error: {{ error }}
  </div>

  <ul v-else>
    <li v-for="product in data" :key="product.ID">
      {{ product.Name }} - {{ product.Price }}
    </li>
  </ul>
</template>
```

## 3. Explore with DevTools

Open your browser and press `Shift + Alt + D` to open Nuxt DevTools. Select the **ODX** tab to:

- **Monitor Traffic:** See live OData requests and responses.
- **Visualize Schemas:** View your entity relationships in a graph.
- **Browse Data:** Inspect raw OData collections in a table view.
