# Introduction

**ODX (OData Developer Experience)** is a modular ecosystem designed to simplify SAP OData integration in Nuxt 4 applications.

## Why ODX?

Standard OData integration often involves repetitive boilerplate and a lack of type safety. ODX automates the friction:

::code-group
  ```bash [pnpm]
  pnpm add @bc8-odx/nuxt
  ```
  ```bash [npm]
  npm install @bc8-odx/nuxt
  ```
  ```bash [yarn]
  yarn add @bc8-odx/nuxt
  ```
::

## Main Components

- **Core (@bc8-odx/core):** The foundation of the toolkit, containing the OData client and type definitions.
- **Proxy (@bc8-odx/proxy):** A Nitro server handler that proxies requests to your SAP backend.
- **Nuxt Module (@bc8-odx/nuxt):** Integrates the core and proxy into Nuxt, providing composables and DevTools.
- **Explorer (@bc8-odx/explorer):** A custom DevTools UI for exploring your OData services.
