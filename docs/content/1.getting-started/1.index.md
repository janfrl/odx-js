# Introduction

**nuxt-sap-odata** is a comprehensive, framework-agnostic toolkit designed to bridge the gap between SAP OData services and modern web applications. While built with a "Nuxt-first" mentality, its core components are decoupled to serve any TypeScript-based environment.

::callout{icon="i-heroicons-light-bulb" color="primary"}
Our primary goal is to provide a **Type-Safe SDK** experience that feels native to modern frontend development, moving away from complex XML configurations.
::

## Key Features

- **🚀 Type-Safe SDK:** Automated TypeScript model generation from EDMX schemas using `odata2ts`.
- **🛠️ First-Class Nuxt Integration:** A dedicated module providing auto-imported composables like `useOData`.
- **🕵️ DevTools UI:** An integrated Explorer for monitoring traffic, visualizing schemas, and browsing entity data.
- **🛡️ Agnostic Proxy:** A powerful server-side proxy layer that handles SAP-specific nuances like CSRF tokens and Basic/Bearer auth.

## Why this toolkit?

Standard OData integration often involves repetitive boilerplate and a lack of type safety. This toolkit automates the friction:

::code-group
  ```bash [pnpm]
  pnpm add nuxt-sap-odata
  ```
  ```bash [npm]
  npm install nuxt-sap-odata
  ```
  ```bash [yarn]
  yarn add nuxt-sap-odata
  ```
::
