# Nuxt Integration (`@bc8-odx/nuxt`)

The Nuxt module is the primary entry point for developers using the ODX ecosystem. It orchestrates the internal proxy, manages automated type generation from EDMX schemas, and provides high-level composables for data fetching.

## Features

- **Type-Safe SDK:** Automatically generates TypeScript models during the Nuxt `prepare` hook.
- **Fluent API:** Deeply integrated `useOData` composable with IDE autocomplete support.
- **Server-Side Proxy:** Transparently handles SAP-specific authentication and CSRF preflights.
- **DevTools Integration:** Native tab in Nuxt DevTools for traffic monitoring and schema visualization.

## Usage

ODX provides a reactive, developer-friendly API that mirrors the structure of your OData services.

::code-group
  ```vue [Basic List]
  <script setup lang="ts">
  // Access services and entity sets using dot-notation
  const { data } = await useOData().V2Service.Products.list({
    $select: ['ID', 'Name', 'Price'],
    $filter: 'Price gt 100.00'
  })
  </script>
  ```

  ```vue [Single Item]
  <script setup lang="ts">
  // Fetch a specific item by its primary key
  const { data: product } = await useOData().V2Service.Products.get('HT-1000')
  </script>
  ```

  ```vue [Dynamic Service]
  <script setup lang="ts">
  // For dynamic service names, use the functional syntax
  const serviceName = 'MyCustomService'
  const { data } = await useOData(serviceName).entitySet('MyEntity').list()
  </script>
  ```
::

## Installation

Add the package to your project:

```bash
pnpm add @bc8-odx/nuxt
```

Then register it in your configuration:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@bc8-odx/nuxt'],
  odata: {
    services: [
      {
        name: 'V2Service',
        url: 'https://services.odata.org/V2/Northwind/Northwind.svc/'
      }
    ]
  }
})
```
