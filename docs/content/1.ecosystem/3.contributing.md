Contributions are welcome! This guide explains how to get the monorepo running locally and how to submit changes.

## Monorepo Structure

ODX is a **pnpm workspace** monorepo. The top-level structure is:

```
nuxt-sap-odata/
├── packages/
│   ├── core/          # @bc8-odx/core
│   ├── nuxt/          # @bc8-odx/nuxt
│   ├── proxy/         # @bc8-odx/proxy
│   └── explorer/      # @bc8-odx/explorer
├── docs/              # This documentation site
├── specs/             # Feature specs and planning documents
└── pnpm-workspace.yaml
```

## Prerequisites

- **Node.js** >= 20
- **pnpm** >= 9

## Setup

```bash
# Clone the repository
git clone https://github.com/janfrl/odx-js.git
cd odx-js

# Install all dependencies across the workspace
pnpm install

# Build all packages (required before running the docs)
pnpm build
```

## Running the Documentation Site

```bash
cd docs
pnpm dev
```

The docs site runs on [http://localhost:3000](http://localhost:3000).

## Running Tests

```bash
# Run tests for all packages
pnpm test

# Run tests for a single package
pnpm --filter @bc8-odx/core test
```

## Submitting Changes

1. Fork the repository and create a feature branch from `main`.
2. Make your changes with clear, focused commits.
3. Ensure `pnpm build` and `pnpm test` pass.
4. Open a pull request with a description of what changed and why.

For documentation-only changes, a build of the docs site is sufficient.
