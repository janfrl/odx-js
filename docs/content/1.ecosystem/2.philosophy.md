## Why OData?

OData (Open Data Protocol) is an OASIS standard that defines a set of best practices for building and consuming RESTful APIs. It is the lingua franca of SAP's enterprise services — from S/4HANA and SuccessFactors to BTP Business Services — making it the most impactful protocol to support well in a modern web toolkit.

## OData V2 and V4 Support

ODX treats both protocol versions as first-class citizens because the SAP ecosystem spans both generations.

::tabs{.w-full}
  :::tabs-item{label="OData V2" icon="i-simple-icons-sap"}
  The dominant protocol in on-premise SAP systems (NetWeaver, ECC, S/4HANA).

  - **Metadata**: XML-based (EDMX)
  - **Auth**: CSRF token protection for state-changing requests.
  - **Quirks**: ODX handles `sap-client` injection and cookie sessions automatically.
  :::

  :::tabs-item{label="OData V4" icon="i-simple-icons-sap"}
  The current standard used by SAP BTP services and S/4HANA Cloud.

  - **Metadata**: JSON-first metadata (CSDL) or XML.
  - **Queries**: Richer query capabilities (`$apply`, `$compute`).
  - **Auth**: Modern OAuth 2.0 patterns; no CSRF tokens required.
  :::
::

## Design Pillars

:::card-group
  ::card
  ---
  title: Security as a Default
  icon: i-heroicons-shield-check
  ---
  Proxied-by-default ensures credentials never leave the server. Includes a declarative policy engine and built-in BTP Security Context support.
  ::

  ::card
  ---
  title: Framework Agnosticism
  icon: i-heroicons-code-bracket
  ---
  Core logic is free of framework dependencies. Use the `$odata` client in React, plain Node.js, or SvelteKit without rewriting your logic.
  ::

  ::card
  ---
  title: Offline-First
  icon: i-heroicons-globe-alt
  ---
  Mock backends for rapid UI development. Export entity data from the Explorer and serve it locally without code changes.
  ::
:::
