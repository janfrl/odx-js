Integrating ODX into your Nuxt 4 project is straightforward. Follow these steps to go from zero to your first type-safe OData request.

:::steps{level="2"}

## Install the Package

Add `@bc8-odx/nuxt` using your preferred package manager.

:::code-group
```bash [pnpm]
pnpm add @bc8-odx/nuxt
```
```bash [npm]
npm install @bc8-odx/nuxt
```
```bash [yarn]
yarn add @bc8-odx/nuxt
```
:::

## Register the Module

Add the module to the `modules` array in your `nuxt.config.ts`.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: [
    '@bc8-odx/nuxt'
  ]
})
```

## Configure Your Services

Add an `odata` block to declare your OData services. The `name` is used as the key in the `useOData()` dot-notation API.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@bc8-odx/nuxt'],

  odata: {
    // Optional: the base path for the internal ODX proxy route
    basePath: '/api/odx',

    services: [
      {
        name: 'V2Service',
        url: 'https://services.odata.org/V2/Northwind/Northwind.svc/',
        strategy: 'proxied'
      }
    ]
  }
})
```

### Service Strategy

| Strategy | Description |
| --- | --- |
| `proxied` (default) | Requests are routed through Nitro. Handles authentication (Basic, Bearer, BTP) and CSRF automatically. Recommended for SAP. |
| `direct` | Requests are sent directly from the browser to the OData backend. Suitable for public APIs or when CORS is already handled. |

### Security Rules

Fine-grained security policies can be applied per service to restrict HTTP methods or block specific paths.

```ts [nuxt.config.ts]
services: [
  {
    name: 'AdminService',
    url: '...',
    rules: [
      { type: 'allowOnlyMethods', value: ['GET'] },
      { type: 'denyPath', value: '/$metadata' }
    ]
  }
]
```

### Configuration Priority

Settings are merged from multiple sources in the following order (highest wins):

1. **Environment Variables**: `NUXT_ODATA_SERVICES_<NAME>_URL`
2. **SAP BTP User-Provided Services**: Runtime overrides via BTP bindings.
3. **Local Configuration**: Settings in `nuxt.config.ts`

## Generate TypeScript Types

Run the `prepare` command to fetch your service metadata and generate TypeScript types.

```bash
npx nuxi prepare
```

This creates a `.odx/` directory in your project root containing the generated models and EDMX metadata. Re-run this command whenever the backend schema changes.

## Fetch Your First Data

You are ready to use the `useOData` composable in any Vue component or page.

```vue [pages/products.vue]
<script setup lang="ts">
const { data, pending, error } = await useOData().V2Service.Products.list({
  $select: ['ProductID', 'ProductName'],
  $top: 10
})
</script>

<template>
  <ul>
    <li v-for="p in data" :key="p.ProductID">{{ p.ProductName }}</li>
  </ul>
</template>
```

:::
