The `@bc8-odx/nuxt` module is configured via the `odata` block in `nuxt.config.ts`.

## Module Options

::api-reference{name="ModuleOptions"}
::

## `odata.basePath`

| Type | Default |
| --- | --- |
| `string` | `'/api/odx'` |

The Nitro route prefix under which the ODX proxy handler is registered. All proxied service requests are forwarded through this path.

## `odata.services`

An array of service definitions. Each entry describes one OData backend.

| Property | Type | Required | Description |
| --- | --- | --- | --- |
| `name` | `string` | Yes | Unique identifier. Used as the key in `useOData()`. |
| `url` | `string` | Yes* | Base URL of the OData service. |
| `destination` | `string` | No | BTP Destination name (overrides `url` on BTP). |
| `strategy` | `'proxied' \| 'direct'` | No | Default: `proxied`. |
| `version` | `'v2' \| 'v4'` | No | OData protocol version. Auto-detected from `$metadata` if omitted. |
| `auth` | `AuthOptions` | No | Static credentials for Basic or Bearer auth. |
| `rules` | `PolicyRule[]` | No | Declarative security policies. |

*Either `url` or `destination` is required.

## `odata.devtools`

Controls the ODX Explorer integration in Nuxt DevTools.

| Property | Type | Default | Description |
| --- | --- | --- | --- |
| `enabled` | `boolean` | `true` | Enable the Explorer panel. |
| `maxLogs` | `number` | `100` | Maximum number of traffic entries to retain. |

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  odata: {
    devtools: {
      enabled: true,
      maxLogs: 200
    }
  }
})
```

## Environment Variable Overrides

Any service property can be overridden at runtime via environment variables:

```bash
NUXT_ODATA_SERVICES_V2SERVICE_URL=https://new-backend.example.com
NUXT_ODATA_SERVICES_V2SERVICE_AUTH_USERNAME=admin
```

The pattern is `NUXT_ODATA_SERVICES_<NAME>_<PROPERTY>` where `<NAME>` is the uppercase service name.
