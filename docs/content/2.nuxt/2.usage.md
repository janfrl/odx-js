ODX provides a reactive, type-safe wrapper around Nuxt's `useFetch` called `useOData`.

## Usage Patterns

:::tabs{.w-full}
  :::tabs-item{label="Dot-Notation" icon="i-lucide-mouse-pointer-2"}
  The recommended way to access your OData services. It provides full IDE autocomplete for service and entity names.

  ```vue [app.vue]
  <script setup lang="ts">
  const { data } = await useOData().V2Service.Products.list({
    $select: ['ProductID', 'ProductName'],
    $filter: 'UnitPrice gt 20'
  })
  </script>
  ```
  :::

  :::tabs-item{label="Functional" icon="i-lucide-code"}
  Use this for dynamic service or entity names (e.g., from route parameters).

  ```ts
  const service = 'V2Service'
  const entity = 'Products'
  const { data } = await useOData(service).entitySet(entity).list()
  ```
  :::
:::

## Available Methods

Every entity set provides the following CRUD methods:

:::field-group
  :::field{name=".list(query?, options?)" type="GET"}
  Fetches a collection of entities.
  :::

  :::field{name=".get(id, query?, options?)" type="GET"}
  Fetches a single entity by its primary key.
  :::

  :::field{name=".create(payload)" type="POST"}
  Creates a new entity.
  :::

  :::field{name=".update(id, payload)" type="PATCH"}
  Updates an existing entity.
  :::

  :::field{name=".remove(id)" type="DELETE"}
  Removes an entity.
  :::
:::

## Query Parameters

The `.list()` and `.get()` methods accept an `ODataQuery` object:

:::field-group
  :::field{name="$select" type="string | string[]"}
  Fields to include in the response.
  :::

  :::field{name="$filter" type="string"}
  OData filter expression (e.g. `Price gt 10`).
  :::

  :::field{name="$orderby" type="string"}
  Sort order (e.g. `Price desc, Name asc`).
  :::

  :::field{name="$top" type="number"}
  Limit the number of results to return.
  :::

  :::field{name="$skip" type="number"}
  Number of results to skip for pagination.
  :::

  :::field{name="$expand" type="string"}
  Navigation properties to include in the response.
  :::
:::
