The `useOData` composable provides a type-safe, reactive wrapper around Nuxt's `useFetch`.

## Usage Patterns

### Dot-Notation
This is the recommended way to access OData services. It provides full IDE autocomplete for service and entity names based on your generated types.

```ts
// Autocomplete for .V2Service and .Products
const { data, pending, error } = await useOData().V2Service.Products.list({
  $select: ['ProductID', 'ProductName'],
  $filter: 'UnitPrice gt 20'
})
```

### Functional Syntax
Use this for dynamic service or entity names (e.g. from route params).

```ts
const service = 'V2Service'
const entity = 'Products'

const { data } = await useOData(service).entitySet(entity).list()
```

---

## Method Reference

| Method | Returns | Description |
| --- | --- | --- |
| `.list(query?, options?)` | `AsyncData<T[]>` | Fetches a collection of entities. |
| `.get(id, query?, options?)` | `AsyncData<T>` | Fetches a single entity by ID. |
| `.create(payload)` | `Promise<T>` | POST request to create an entity. |
| `.update(id, payload)` | `Promise<T>` | PATCH request to update an entity. |
| `.remove(id)` | `Promise<void>` | DELETE request to remove an entity. |

---

## Query Parameters

The `.list()` and `.get()` methods accept an `ODataQuery` object:

| Parameter | Type | Description |
| --- | --- | --- |
| `$select` | `string[]` | Specific properties to return. |
| `$filter` | `string` | OData filter expression. |
| `$orderby` | `string` | Sort order (e.g. `Price desc`). |
| `$top` | `number` | Number of results to return. |
| `$skip` | `number` | Offset for pagination. |
| `$expand` | `string` | Navigation properties to include. |
