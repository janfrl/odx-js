The `useOData` composable is auto-imported by the `@bc8-odx/nuxt` module. It provides a type-safe, reactive wrapper around Nuxt's `useFetch`.

## Usage Patterns

### Dot-Notation (Recommended)

Provides full IDE autocomplete for service and entity names based on your generated types.

```ts
const { data, pending, error } = await useOData().V2Service.Products.list({
  $select: ['ProductID', 'ProductName'],
  $filter: 'UnitPrice gt 20'
})
```

### Functional Syntax

Use for dynamic service or entity names (e.g. from route params).

```ts
const service = 'V2Service'
const entity = 'Products'

const { data } = await useOData(service).entitySet(entity).list()
```

## Method Reference

| Method | HTTP | Returns | Description |
| --- | --- | --- | --- |
| `.list(query?, options?)` | `GET` | `AsyncData<T[]>` | Fetches a collection of entities. |
| `.get(id, query?, options?)` | `GET` | `AsyncData<T>` | Fetches a single entity by ID. |
| `.create(payload)` | `POST` | `Promise<T>` | Creates a new entity. |
| `.update(id, payload)` | `PATCH` | `Promise<T>` | Updates an existing entity. |
| `.remove(id)` | `DELETE` | `Promise<void>` | Removes an entity. |

---

## Query Parameters (`ODataQuery`)

Accepted by `.list()` and `.get()`.

| Parameter | Type | Description |
| --- | --- | --- |
| `$select` | `string[]` | Specific properties to return. |
| `$filter` | `string` | OData filter expression. |
| `$orderby` | `string` | Sort order (e.g. `Price desc`). |
| `$top` | `number` | Number of results to return. |
| `$skip` | `number` | Offset for pagination. |
| `$expand` | `string` | Navigation properties to include. |

---

## Return Type (`ODataAsyncData<T>`)

`useOData` returns the same shape as Nuxt's `AsyncData`:

| Property | Type | Description |
| --- | --- | --- |
| `data` | `Ref<T \| null>` | The reactive response payload. |
| `pending` | `Ref<boolean>` | `true` while the request is in-flight. |
| `error` | `Ref<Error \| null>` | The error object if the request failed. |
| `status` | `Ref<AsyncDataRequestStatus>` | One of `'idle'`, `'pending'`, `'success'`, `'error'`. |
| `refresh()` | `() => Promise<void>` | Re-runs the request with the same parameters. |
