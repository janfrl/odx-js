Configure your OData services in the `odata` block of your `nuxt.config.ts`.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@bc8-odx/nuxt'],

  odata: {
    // Prefix for the internal proxy routes
    basePath: '/api/odx',

    services: [
      {
        name: 'V2Service',
        url: 'https://services.odata.org/V2/Northwind/Northwind.svc/',
        strategy: 'proxied',
        rules: [
          { type: 'allowOnlyMethods', value: ['GET'] }
        ]
      }
    ]
  }
})
```

## Global Options

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `basePath` | `string` | `/api/odx` | The URL prefix where the proxy handlers are mounted. |
| `mode` | `'sdk'` | `sdk` | The generation mode. |
| `btpConfigService` | `string` | `odx-config` | Name of the BTP User-Provided Service for runtime overrides. |

## Service Options

Each service in the `services` array supports the following configuration:

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `name` | `string` | - | **Required.** Unique identifier for the service. |
| `url` | `string` | - | **Required.** URL to the service root or path to an EDMX file. |
| `strategy` | `'proxied' \| 'direct'` | `proxied` | `proxied` uses the server handler; `direct` fetches from the client. |
| `route` | `string` | name | Custom route segment for the proxy URL. |
| `auth` | `object` | - | Optional credentials (username, password, bearerToken). |
| `rules` | `ODataRule[]` | `[]` | List of security and transformation rules. See [Proxy Policies](/proxy/policies). |

## Configuration Priority

The ODX module merges configuration from multiple sources. If a setting is defined in multiple places, the higher priority source wins:

1. **Environment Variables**: e.g., `NUXT_ODATA_SERVICES_MYSERVICE_URL`. (Highest priority)
2. **SAP BTP Runtime Overrides**: Configuration provided via the `odx-config` service binding.
3. **Local Configuration**: Settings defined in `nuxt.config.ts`. (Lowest priority)

This hierarchy allows developers to provide sensible defaults while letting administrators adjust production settings without code changes.
