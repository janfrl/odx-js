Configure your OData services in the `odata` block of your `nuxt.config.ts`.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@bc8-odx/nuxt'],

  odata: {
    // Prefix for the internal proxy routes
    basePath: '/api/odx',

    services: [
      {
        name: 'V2Service',
        url: 'https://services.odata.org/V2/Northwind/Northwind.svc/',
        strategy: 'proxied'
      },
      {
        name: 'V4Service',
        url: 'https://services.odata.org/V4/Northwind/Northwind.svc/'
      }
    ]
  }
})
```

::callout{icon="i-heroicons-information-circle" color="info"}
You can override or extend these settings using environment variables prefixed with `NUXT_ODATA_SERVICES_`.
::

## Global Options

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `basePath` | `string` | `/api/odx` | The URL prefix where the proxy handlers are mounted. |
| `mode` | `'sdk'` | `sdk` | The generation mode. Currently only `sdk` is supported. |

## Service Options

Each service in the `services` array supports the following configuration:

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `name` | `string` | - | **Required.** Unique identifier for the service. |
| `url` | `string` | - | **Required.** URL to the service root or path to an EDMX file. |
| `strategy` | `'proxied' \| 'direct'` | `proxied` | `proxied` uses the server handler; `direct` fetches from the client. |
| `route` | `string` | name | Custom route segment for the proxy URL. |
| `auth` | `object` | - | Optional credentials (username, password, bearerToken). |
