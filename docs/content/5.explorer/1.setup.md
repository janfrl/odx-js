The ODX Explorer is your primary tool for debugging OData services during development. It is embedded directly in **Nuxt DevTools** and requires no additional installation when using `@bc8-odx/nuxt`.

::note
The Explorer is automatically available when `@bc8-odx/nuxt` is registered in `modules` and `devtools: { enabled: true }` is set in the `odata` block (default).
::

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  odata: {
    devtools: {
      enabled: true,
      maxLogs: 100
    }
  }
})
```

::steps{level="2"}

## Open Nuxt DevTools

Start your Nuxt application in development mode:

```bash
pnpm run dev
```

Open your browser and press `Shift + Alt + D` or click the Nuxt logo at the bottom of the screen.

## Locate the ODX Tab

In the DevTools sidebar, look for the **ODX** icon. Clicking it opens the Explorer interface.

## Explore Schemas

Select one of your configured services from the dropdown menu. You will see a list of available entity sets. Clicking on an entity set reveals its properties, key fields, and navigation associations.

## Monitor Requests

Trigger an OData request in your application (e.g., by navigating to a page that uses `useOData`). Switch to the **Traffic** tab in the Explorer to see the request URL, headers, payload, and backend response.

## Export Mock Data

In the **Data Browser** tab, you can download entity data as JSON. Place the exported files in `.odx/mock/` to use them for offline development.

::
