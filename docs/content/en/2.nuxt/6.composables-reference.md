The `useOData` composable is auto-imported by the `@me-tools/odx-nuxt` module. It provides a type-safe, reactive wrapper around Nuxt's `useFetch`.

## Usage Patterns

### Dot-Notation (Recommended)

Provides full IDE autocomplete for service and entity names based on your generated types.

```ts
const { data, pending, error } = await useOData().V2Service.Products.list({
  $select: ['ProductID', 'ProductName'],
  $filter: 'UnitPrice gt 20'
})
```

### Functional Syntax

Use for dynamic service or entity names (e.g. from route params).

```ts
const service = 'V2Service'
const entity = 'Products'

const { data } = await useOData(service).entitySet(entity).list()
```

## Method Reference

| Method | HTTP | Returns | Description |
| --- | --- | --- | --- |
| `.list(query?, options?)` | `GET` | `AsyncData<T[]>` | Fetches a collection of entities. |
| `.fetchList(query?, options?)` | `GET` | `Promise<T[]>` | Performs an imperative collection read. |
| `.get(id, query?, options?)` | `GET` | `AsyncData<T>` | Fetches a single entity by ID. |
| `.create(payload)` | `POST` | `Promise<T>` | Creates a new entity. |
| `.update(id, payload)` | `PATCH` | `Promise<T>` | Updates an existing entity. |
| `.remove(id)` | `DELETE` | `Promise<void>` | Removes an entity. |

Use `.list()` in component or page setup for Nuxt SSR-aware `AsyncData`.
Use `.fetchList()` for controller events and other imperative effects. Request
options such as an `AbortSignal` are forwarded to the ODX transport.
