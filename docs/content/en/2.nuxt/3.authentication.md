ODX supports various authentication methods for connecting to SAP and non-SAP backends. All credentials are managed server-side by the proxy — they are never exposed to the browser.

## Authentication Methods

:::accordion
  :::accordion-item{label="Basic Authentication" icon="i-heroicons-user"}
  Provide credentials directly in the service configuration. Ideal for development against NetWeaver systems.

  ```ts [nuxt.config.ts]
  services: [
    {
      name: 'V2Service',
      auth: {
        username: 'USER',
        password: 'PASSWORD'
      }
    }
  ]
  ```
  :::

  :::accordion-item{label="Bearer Token" icon="i-heroicons-key"}
  For systems using OAuth 2.0, you can pass a static bearer token.

  ```ts [nuxt.config.ts]
  auth: {
    bearerToken: 'ey...'
  }
  ```
  :::

  :::accordion-item{label="SAP BTP Principal Propagation" icon="i-simple-icons-sap"}
  When deployed to SAP BTP, ODX handles principal propagation automatically via the Destination and XSUAA services.

  ::steps{level="3"}
  ### Define the Destination
  Specify the `destination` name in your config (must match the name in BTP Cockpit).

  ```ts [nuxt.config.ts]
  services: [
    {
      name: 'S4Products',
      destination: 'S4_BACKEND_DEST'
    }
  ]
  ```

  ### Bind Services
  Ensure your application is bound to the **Destination** and **XSUAA** services in BTP.

  ### Identity Propagation
  ODX uses `@sap-cloud-sdk/connectivity` to fetch the token and propagate the user's identity at runtime.
  ::
  :::
:::

## CSRF Token Handling

SAP systems often require a CSRF token for state-changing requests (`POST`, `PATCH`, `DELETE`). ODX manages this automatically:

:::steps{level="3"}
  ### Pre-fetch
  Before a state-changing request, ODX fetches a CSRF token via a `HEAD` or `GET` request.

  ### Cache
  The token is cached and reused for subsequent requests in the same session.

  ### Auto-Refresh
  If a token is rejected (e.g., after session expiry), ODX automatically refreshes it and retries the request.
:::

No configuration is required — CSRF handling is active by default for all proxied services.
