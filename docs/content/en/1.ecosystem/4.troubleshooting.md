This guide provides solutions for common problems you might encounter while using ODX.

:::accordion

:::accordion-item{label="1. Type Generation Fails" icon="i-lucide-file-warning"}
If `npx nuxi prepare` does not generate types for your services:

- **Check the URL**: Ensure the `url` in `nuxt.config.ts` is correct and accessible.
- **EDMX Accessibility**: If the URL points to a live service, ODX needs to be able to fetch the `$metadata` without authentication during the `prepare` step (or you must provide credentials).
- **Local EDMX**: Try downloading the EDMX file and pointing the `url` to the local path.
- **Stale metadata**: Delete `.nuxt/odx/temp/` to clear generated Nuxt state. If ODX still reuses old metadata, delete the matching `.odx/cache/<service>.edmx` fallback file so the next prepare run refetches it.
:::

:::accordion-item{label="2. Authentication Errors (401/403)" icon="i-lucide-lock"}
- **Proxy Strategy**: Ensure you are using `strategy: 'proxied'` if you need ODX to handle Basic/Bearer/BTP auth.
- **BTP Destinations**: Verify that the destination name matches exactly what is in BTP Cockpit.
- **Principal Propagation**: Ensure the user is logged into the Nuxt app via XSUAA.
:::

:::accordion-item{label="3. CSRF Token Issues" icon="i-lucide-shield-check"}
- **HEAD Requests**: Some SAP systems require `GET` instead of `HEAD` for CSRF. Check your backend configuration.
- **Proxy Logs**: Enable DevTools logs to see if CSRF pre-fetching is succeeding.
:::

:::accordion-item{label="4. Explorer Not Showing" icon="i-lucide-eye-off"}
- **Dev Mode**: The Nuxt DevTools Explorer only works in development mode (`nuxt dev`).
- **Module Order**: Ensure `@bc8-odx/nuxt` is registered in `modules`.
- **DevTools Enabled**: Verify that `devtools: { enabled: true }` is set in the `odata` block.
- **Production Explorer**: In BTP, use the standalone Explorer behind the
  AppRouter. `/__odx__/client/*` should route to the Explorer UI, while only
  `/__odx__/{config,logs,schema,generate,types,me}` should route to the proxy.
:::

:::accordion-item{label="5. Production Explorer Logs Are Empty" icon="i-lucide-list-x"}
Production `/__odx__/logs` returns an empty list and rejects clearing with
`403` unless SQL log storage is explicitly configured. Enable the proxy-owned
db0-backed store with `NUXT_ODATA_DEVTOOLS_LOG_STORE=sql` and a managed SQL
database connection when deployed traffic history is required. Stored
production entries remain redacted, request and response payload bodies are
omitted by default, and the Explorer never connects to the database directly.
:::

:::accordion-item{label="6. Proxy Returns 502 Bad Gateway" icon="i-lucide-server-off"}
- **Backend URL**: Confirm the backend URL is reachable from the server where Nitro runs.
- **On-Premise Systems**: If the backend is on-premise, ensure the BTP Connectivity Service is bound and the destination is configured with `ProxyType: OnPremise`.
:::
