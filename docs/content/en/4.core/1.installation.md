`@me-tools/odx-core` is a framework-agnostic package. Use it when you need standard OData TypeScript interfaces or the lightweight `$odata` fetch client outside of a Nuxt application.

:::steps{level="2"}

## Install the Package

:::code-group
```bash [pnpm]
pnpm add @me-tools/odx-core
```
```bash [npm]
npm install @me-tools/odx-core
```
```bash [yarn]
yarn add @me-tools/odx-core
```
:::

## Import Types

Use the shared OData types in any TypeScript project.

```ts
import type { ODataQuery, ODataResponse } from '@me-tools/odx-core'

const query: ODataQuery = {
  $select: ['ProductID', 'ProductName'],
  $filter: 'UnitPrice gt 20',
  $top: 10
}
```

## Use the `$odata` Client

The `$odata` client is a lightweight, promise-based HTTP client for OData services.

```ts
import { $odata } from '@me-tools/odx-core'

const products = await $odata('https://services.odata.org/V2/Northwind/Northwind.svc/')
  .entitySet('Products')
  .list({ $top: 5 })
```

:::

## When to Use Core Directly

| Situation | Recommended Package |
| --- | --- |
| Building a Nuxt 4 application | [`@me-tools/odx-nuxt`](/nuxt/getting-started) |
| Building a BFF proxy layer | [`@me-tools/odx-proxy`](/proxy/installation) |
| Writing shared OData types or utilities | `@me-tools/odx-core` |
| Integrating OData in a React/Plain Node app | `@me-tools/odx-core` |
