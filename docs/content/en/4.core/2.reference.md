`@me-tools/odx-core` is a framework-agnostic package containing the core OData logic, shared TypeScript types, and the base `$odata` client.

## Key Interfaces

### `ODataQuery`

Used to define structured queries for OData services.

| Property | Type | Description |
| --- | --- | --- |
| `$select` | `string \| string[]` | Fields to include in the response. |
| `$filter` | `string` | OData filter expression. |
| `$orderby` | `string` | Sort order. |
| `$top` | `number` | Limit result count. |
| `$skip` | `number` | Offset for pagination. |
| `$expand` | `string` | Navigation properties to include. |

### `ODataResponse<T>`

The normalized response envelope returned by the `$odata` client.

| Property | Type | Description |
| --- | --- | --- |
| `value` | `T[]` | The entity collection. |
| `@odata.count` | `number \| undefined` | Total count when `$count=true` is requested. |
| `@odata.nextLink` | `string \| undefined` | URL for server-side pagination. |

## The `$odata` Client

A lightweight, promise-based client that can be used in any TypeScript environment.

```ts
import { $odata } from '@me-tools/odx-core'

// List with query options
const products = await $odata('https://.../V2Service')
  .entitySet('Products')
  .list({ $select: ['ProductID'], $top: 5 })

// Get by key
const product = await $odata('https://.../V2Service')
  .entitySet('Products')
  .get(1)

// Create
const created = await $odata('https://.../V2Service')
  .entitySet('Products')
  .create({ ProductName: 'New Product' /* ... */ })
```

## Internal Utilities

The package also exports utilities used internally by `@me-tools/odx-proxy` and `@me-tools/odx-nuxt`:

| Export | Description |
| --- | --- |
| buildODataUrl | Serializes an ODataQuery into a URL query string. |
| parseCsrfToken | Extracts the CSRF token from a response header. |
| normalizeSapHeaders | Applies SAP-specific header defaults (sap-client, X-Requested-With). |
