Integrating SAP OData services into modern JavaScript and TypeScript applications has historically been a journey filled with friction. Developers often find themselves wrestling with **CORS issues**, **magic string URLs**, **manual authentication handling**, and **untyped JSON payloads** that break silently.

**ODX (OData Developer Experience)** was built to transform this process. It provides a modular, type-safe, and visually intuitive toolkit that makes working with OData feel like using a modern SDK rather than a legacy XML protocol.

## The ODX Ecosystem

ODX is not just a single library; it is a carefully crafted ecosystem of four interlocking packages designed to handle every stage of the OData lifecycle.

::list{type="primary"}
- **@bc8-odx/core**: The foundation. It provides framework-agnostic TypeScript interfaces, recursive data flattening utilities, and a lightweight OData client.
- **@bc8-odx/proxy**: The interceptor. A powerful H3 server handler that manages SAP-specific nuances like CSRF token preflights, Basic/Bearer auth, and local JSON mocking.
- **@bc8-odx/nuxt**: The glue. The official Nuxt module that automates type generation, registers the proxy, and provides the elegant `useOData()` composables.
- **@bc8-odx/explorer**: The visualizer. A high-fidelity DevTools UI for monitoring live OData traffic, inspecting entity relationships, and exploring data collections.
::

::callout{icon="i-heroicons-sparkles" color="primary"}
Whether you are building a complex enterprise dashboard or a lightweight microservice, ODX scales with you, providing the right tool for the job.
::
