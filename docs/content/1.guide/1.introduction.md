Integrating SAP OData into modern TypeScript applications is often filled with friction—magic strings, manual auth handling, and untyped payloads. **ODX (OData Developer Experience)** transforms this into a type-safe, modular, and visually intuitive workflow.

## The Ecosystem

ODX is a collection of four interlocking packages:

- **@bc8-odx/core**: Framework-agnostic types and OData utilities.
- **@bc8-odx/proxy**: H3 server handler for auth, CSRF, and mocking.
- **@bc8-odx/nuxt**: The official Nuxt module for auto-types and composables.
- **@bc8-odx/explorer**: A high-fidelity DevTools UI for traffic and schema analysis.

## Core Philosophies

- **Type Safety First**: Automated model generation from your actual SAP schema.
- **Agnostic Heart**: Core logic works in any TypeScript environment, not just Nuxt.
- **Offline Ready**: Toggle between live backends and local JSON mocks seamlessly.
