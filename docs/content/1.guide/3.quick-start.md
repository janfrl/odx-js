Get up and running with ODX. Choose the path that matches your architecture.

::code-group
  ```md [Nuxt (Full-Stack)]
  # Quick Start for Nuxt

  Assume you have installed `@bc8-odx/nuxt`.

  ### 1. Generate Types
  ODX generates TypeScript models from your EDMX schema.
  ```bash
  npx nuxi prepare
  ```

  ### 2. Fetch Data
  Use the `useOData` composable in your components.
  ```vue
  <script setup lang="ts">
  const { data } = await useOData().MyService.Products.list()
  </script>
  ```

  ### 3. Debug with DevTools
  Press `Shift + Alt + D` to see live traffic and schema graphs.
  ```

  ```md [Proxy (Middleware)]
  # Quick Start for Proxy

  Assume you have installed `@bc8-odx/proxy`.

  ### 1. Create the Handler
  Mount the OData proxy on your server.
  ```ts
  import { createODataHandler } from '@bc8-odx/proxy'
  const handler = createODataHandler({
    services: [{ name: 'Northwind', url: '...' }]
  })
  ```

  ### 2. Test the Endpoint
  Your proxy now serves OData at the mounted path.
  ```bash
  curl http://localhost:3000/api/odata/Northwind/Products
  ```

  ### 3. Benefits
  - Automatic CSRF Token preflights.
  - Basic/Bearer Auth abstraction.
  - Optional JSON Mocking.
  ```
::

::list{type="success"}
- **Productivity:** Stop writing raw fetch calls and manual headers.
- **Type Safety:** Ensure your frontend matches your SAP schema perfectly.
- **Independence:** Mock your backend to develop even when SAP is offline.
::
