Get up and running with ODX in your Nuxt application. This guide assumes you have already [installed the module](/guide/installation).

## 1. Configure your Service

Define your OData backend in your `nuxt.config.ts`. You can add multiple services here.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@bc8-odx/nuxt'],
  odata: {
    services: [
      {
        name: 'V2Service',
        url: 'https://services.odata.org/V2/Northwind/Northwind.svc/'
      }
    ]
  }
})
```

## 2. Generate TypeScript Types

ODX needs to read your service schema to provide autocomplete. Run the prepare command to generate the SDK models.

```bash
npx nuxi prepare
```

## 3. Fetch Data in your Component

Use the `useOData()` composable to fetch data. The syntax is designed to be fluent and highly discoverable via your IDE's autocomplete.

```vue [pages/products.vue]
<script setup lang="ts">
// Dot-notation provides autocomplete for services and their entity sets
const { data, pending, error } = await useOData().V2Service.Products.list({
  $select: ['ID', 'Name', 'Price'],
  $filter: "Price gt 50.00"
})
</script>

<template>
  <main>
    <h1>Products</h1>

    <div v-if="pending">Fetching from SAP...</div>

    <ul v-else-if="data">
      <li v-for="product in data" :key="product.ID">
        {{ product.Name }} - {{ product.Price }}
      </li>
    </ul>

    <div v-else-if="error">
      Something went wrong: {{ error.message }}
    </div>
  </main>
</template>
```
