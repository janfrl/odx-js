ODX is designed to be flexible and can be integrated into various environments, from full-scale Nuxt applications to standalone backend services.

## Choose Your Path

::code-group
  ```bash [Nuxt Module (Recommended)]
  pnpm add @bc8-odx/nuxt
  ```
  ```bash [Standalone Proxy]
  pnpm add @bc8-odx/proxy
  ```
  ```bash [Core Types Only]
  pnpm add @bc8-odx/core
  ```
::

---

### Nuxt Module
The Nuxt integration is the most comprehensive way to use ODX. It automatically sets up the server-side proxy, handles type generation, and injects the DevTools.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@bc8-odx/nuxt'],
  odata: {
    services: [
      {
        name: 'Northwind',
        url: 'https://services.odata.org/V2/Northwind/Northwind.svc/'
      }
    ]
  }
})
```

### Standalone Proxy
If you are building a backend service using **H3** or **Express**, you can use the proxy layer in isolation to handle CSRF and authentication preflights.

```ts [server.ts]
import { createApp, createRouter, eventHandler } from 'h3'
import { createODataHandler } from '@bc8-odx/proxy'

const app = createApp()
const router = createRouter()

// Mount the OData handler on your preferred route
router.use('/api/odata/**', eventHandler(createODataHandler(config)))
```

### Core Types
For projects that only require standardized OData V2/V4 TypeScript interfaces and utility functions (like data flattening), simply install the core package.

```ts [types.ts]
import type { ODataAsyncData } from '@bc8-odx/core'
```

---

::callout{icon="i-heroicons-exclamation-triangle" color="warning"}
**ODX is a pure ESM package.** Ensure your project is configured with `"type": "module"` in your `package.json` to avoid resolution issues.
::
