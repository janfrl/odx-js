The ODX Proxy includes a powerful, **auto-traced Policy Engine** that allows you to enforce security rules and transform requests using a clean, declarative API.

## Overview

Policies are defined using Nitro plugins. By using the `odataGuard` utility, every rule you define is automatically recorded in the ODX Explorer's **Proxy Telemetry**, providing full visibility into why requests are allowed, altered, or denied.

```typescript
import { odataGuard } from '@bc8-odx/proxy'

export default defineNitroPlugin((nitro) => {
  // Global policy
  nitro.hooks.hook('odx:proxy:request', (ctx) => {
    odataGuard(ctx).info('Global security check active')
  })

  // Service-specific policy
  nitro.hooks.hook('odx:proxy:request:MyService', (ctx) => {
    odataGuard(ctx)
      .allowOnlyMethods(['GET', 'POST'])
      .requireScope('Display')
  })
})
```

## Security Helpers

### `requireScope(scope: string)`
Verifies that the current user has the specified XSUAA scope. If the scope is missing, the request is rejected with a `403 Forbidden` and the failure is logged in the telemetry.

### `requireAttribute(name: string, value: string)`
Ensures a specific user attribute (e.g., `company_id`) in the identity token matches the required value.

### `denyIfPathIncludes(path: string, reason?: string)`
Blocks access to specific entity sets or sub-paths.
*   **Example**: `.denyIfPathIncludes('/Users', 'User management is restricted')`

## Method Control

### `allowOnlyMethods(methods: string[])`
Restricts the service to a specific set of HTTP methods.
*   **Example**: `.allowOnlyMethods(['GET'])` makes a service effectively read-only.

### `denyMethods(methods: string[])`
Specifically blocks certain methods while allowing others.

## Transformation Helpers

### `injectHeader(name: string, value: string)`
Adds a custom header to the outgoing request to the SAP backend. This is useful for passing technical identifiers or API keys.

### `rewritePath(pattern: string | RegExp, replacement: string)`
Dynamically modifies the request URL before it is forwarded.
*   **Example**: `.rewritePath(/^\/v2/, '/sap/opu/odata/sap')`

## Custom Logic

### `validate(name: string, callback: fn)`
For complex logic that isn't covered by standard helpers. The callback receives the full request context.
*   **Example**:
    ```typescript
    odataGuard(ctx).validate('BusinessHours', () => {
      const hour = new Date().getHours()
      return hour >= 8 && hour <= 18
    }, 'Outside of business hours')
    ```

### `info(message: string, details?: any)`
Adds a non-blocking informational entry to the proxy trace. Use this to document custom logic steps without affecting the request flow.
