ODX provides a reactive, type-safe wrapper around Nuxt's `useFetch` called `useOData`.

## The Dot-Notation Syntax

This is the recommended way to access your OData services. It provides full IDE autocomplete for service and entity names based on the types generated from your EDMX metadata.

```vue [app.vue]
<script setup lang="ts">
// Autocomplete for .V2Service and .Products
const { data, pending, error } = await useOData().V2Service.Products.list({
  $select: ['ProductID', 'ProductName'],
  $filter: 'UnitPrice gt 20',
  $orderby: 'ProductName asc'
})
</script>

<template>
  <div v-if="data">
    <div v-for="product in data" :key="product.ProductID">
      {{ product.ProductName }}
    </div>
  </div>
</template>
```

## Functional Syntax

If you need to access services or entity sets dynamically (e.g., based on route parameters), use the functional syntax.

```ts
const service = 'V2Service'
const entity = 'Products'

const { data } = await useOData(service).entitySet(entity).list()
```

## Available Methods

Every entity set provides the following CRUD methods:

-   `.list(query?, options?)`: Fetches a collection of entities.
-   `.get(id, query?, options?)`: Fetches a single entity by its primary key.
-   `.create(payload)`: Sends a `POST` request to create a new entity.
-   `.update(id, payload)`: Sends a `PATCH` request to update an existing entity.
-   `.remove(id)`: Sends a `DELETE` request to remove an entity.

## Query Parameters

The `.list()` and `.get()` methods accept an `ODataQuery` object supporting standard OData system query options:

-   `$select`: String or array of fields.
-   `$filter`: OData filter expression.
-   `$top`: Limit results.
-   `$skip`: Pagination offset.
-   `$expand`: Include navigation properties.
