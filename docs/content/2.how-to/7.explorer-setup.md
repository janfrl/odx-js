---
title: Explorer Setup
description: How to use the ODX Explorer in your development workflow.
---

The ODX Explorer is your primary tool for debugging OData services during development.

## 1. Open Nuxt DevTools

Start your Nuxt application in development mode:

```bash
pnpm run dev
```

Open your browser and click on the Nuxt logo at the bottom of the screen or press `Shift + Alt + D`.

## 2. Locate ODX Tab

In the DevTools sidebar, look for the **ODX** icon. Clicking it will open the Explorer interface.

## 3. Explore Schemas

Select one of your configured services from the dropdown menu. You will see a list of available entity sets. Clicking on an entity set will reveal its properties and associations.

## 4. Monitor Requests

Trigger an OData request in your application (e.g., by navigating to a page that uses `useOData`). Switch to the **Traffic** tab in the Explorer to see the request details.

## 5. Mocking Data

The Explorer also allows you to download entity data as JSON, which can then be used for [Offline Mocking](/explanation/proxy-policies).
