ODX allows you to configure multiple OData services with different strategies (proxied or direct) and security rules.

## Basic Setup

Add an `odata` block to your `nuxt.config.ts`. The most critical setting is the `services` array.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@bc8-odx/nuxt'],

  odata: {
    // Optional: The base path for the ODX proxy
    basePath: '/api/odx',

    services: [
      {
        name: 'V2Service',
        url: 'https://services.odata.org/V2/Northwind/Northwind.svc/',
        strategy: 'proxied'
      }
    ]
  }
})
```

## Service Strategy

-   **`proxied` (Default)**: All requests are routed through the Nitro server. This handles authentication (Basic, Bearer, BTP) and CSRF protection automatically. Recommended for most SAP use cases.
-   **`direct`**: Requests are sent directly from the browser to the OData backend. Useful for public APIs or when CORS is already handled.

## Security Rules

You can add fine-grained security policies to each service to restrict methods or paths.

```ts [nuxt.config.ts]
services: [
  {
    name: 'AdminService',
    url: '...',
    rules: [
      { type: 'allowOnlyMethods', value: ['GET'] },
      { type: 'denyPath', value: '/$metadata' }
    ]
  }
]
```

## Configuration Priority

Settings are merged from multiple sources in the following order:

1.  **Environment Variables**: `NUXT_ODATA_SERVICES_<NAME>_URL` (Highest)
2.  **SAP BTP User-Provided Services**: Runtime overrides via BTP bindings.
3.  **Local Configuration**: Settings in `nuxt.config.ts` (Lowest)

## Next Steps

Once configured, run `npx nuxi prepare` again to generate the types for your services, and then you are ready to [Make your first request](/how-to/usage).
