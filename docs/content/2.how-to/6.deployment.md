Deploying an ODX application to SAP BTP involves preparing your `mta.yaml` and ensuring the necessary services are bound.

## 1. Prepare your Project

Ensure your project is a monorepo or has the `@bc8-odx/proxy` logic accessible during build.

## 2. The `mta.yaml` Structure

Your `mta.yaml` should include:
-   **Nuxt Module**: The built Nuxt application.
-   **Managed HTML5 Repository**: If using standalone approuter.
-   **Destination Service**: To resolve OData backends.
-   **XSUAA Service**: For authentication and principal propagation.

```yaml [mta.yaml]
modules:
  - name: my-odx-app
    type: nodejs
    path: .
    requires:
      - name: my-destination-service
      - name: my-xsuaa-service
```

## 3. Build and Deploy

Run the build and deploy commands:

```bash
pnpm run build

pnpm run deploy
```

## 4. Troubleshooting

If your backend calls fail after deployment:
-   Verify that the destination name in `nuxt.config.ts` matches the one in BTP Cockpit.
-   Check the logs of your application in Cloud Foundry using `cf logs my-odx-app --recent`.
