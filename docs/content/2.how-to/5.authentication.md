---
title: Authentication
description: How to handle authentication in ODX.
---

ODX supports various authentication methods for connecting to SAP and non-SAP backends.

## Basic Authentication

You can provide credentials directly in the service configuration.

```ts [nuxt.config.ts]
services: [
  {
    name: 'V2Service',
    url: '...',
    auth: {
      username: 'USER',
      password: 'PASSWORD'
    }
  }
]
```

## Bearer Token

For modern systems using OAuth2, you can pass a static bearer token.

```ts [nuxt.config.ts]
auth: {
  bearerToken: 'ey...'
}
```

## SAP BTP Principal Propagation

When deployed to SAP BTP, ODX automatically handles principal propagation if a `destination` is specified and the app is bound to the Destination and XSUAA services.

1.  Specify the `destination` name in your config.
2.  ODX will use `@sap-cloud-sdk/connectivity` to fetch the token and destination details.
3.  The user's identity is propagated to the backend system.

## CSRF Token Handling

SAP systems often require a CSRF token for state-changing requests (POST, PATCH, DELETE). ODX manages this automatically:

1.  Before a state-changing request, ODX fetches a CSRF token via a `HEAD` or `GET` request.
2.  The token is cached and reused for subsequent requests.
3.  If a token expires, ODX automatically refreshes it.
