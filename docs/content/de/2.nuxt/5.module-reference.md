Das Modul `@bc8-odx/nuxt` wird über den `odata`-Block in der `nuxt.config.ts` konfiguriert.

## Modul-Optionen

::api-ref
---
name: ModuleOptions
---
::

## `odata.basePath`

| Typ | Standard |
| --- | --- |
| `string` | `'/api/odx'` |

Das Nitro-Routen-Präfix, unter dem der ODX Proxy-Handler registriert wird. Alle geproxten Service-Anfragen werden über diesen Pfad weitergeleitet.

## `odata.services`

Ein Array von Service-Definitionen. Jeder Eintrag beschreibt ein OData-Backend.

| Eigenschaft | Typ | Erforderlich | Beschreibung |
| --- | --- | --- | --- |
| `name` | `string` | Ja | Eindeutige Kennung. Wird als Schlüssel in `useOData()` verwendet. |
| `url` | `string` | Ja* | Basis-URL des OData-Services. |
| `destination` | `string` | Nein | BTP Destination-Name (überschreibt `url` auf BTP). |
| `strategy` | `'proxied' \| 'direct'` | Nein | Standard: `proxied`. |
| `version` | `'v2' \| 'v4'` | Nein | OData-Protokollversion. Wird automatisch aus `$metadata` erkannt, wenn weggelassen. |
| `auth` | `AuthOptions` | Nein | Statische Anmeldedaten für Basic- oder Bearer-Authentifizierung. |
| `rules` | `PolicyRule[]` | Nein | Deklarative Sicherheitsrichtlinien. |

*Entweder `url` oder `destination` ist erforderlich.

## `odata.devtools`

Steuert die Integration des ODX Explorers in die Nuxt DevTools waehrend der
lokalen Entwicklung. Diese Optionen aktivieren keine Produktions-Traffic-Logs
und umgehen nicht die Produktions-Authentifizierung und Redaction-Policy fuer
`/__odx__`.

| Eigenschaft | Typ | Standard | Beschreibung |
| --- | --- | --- | --- |
| `enabled` | `boolean` | `true` | Aktiviert das Explorer-Panel. |
| `maxLogs` | `number` | `100` | Maximale Anzahl der zu behaltenden Traffic-Einträge. |

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  odata: {
    devtools: {
      enabled: true,
      maxLogs: 200
    }
  }
})
```

Entwicklungs-Logeintraege sind Diagnosedaten. Redigieren Sie Secrets in
ausgehenden Headern, Auth-/Session-/CSRF-Werten sowie Request- oder
Response-Bodies, und begrenzen Sie grosse Payloads, bevor Sie sie exportieren
oder kopieren. Produktions-Explorer-Logs sind derzeit deaktiviert, bis
persistente Speicherung und Redaction-Policy implementiert sind.

## Overrides über Umgebungsvariablen

Jede Service-Eigenschaft kann zur Laufzeit über Umgebungsvariablen überschrieben werden:

```bash
NUXT_ODATA_SERVICES_V2SERVICE_URL=https://new-backend.example.com
NUXT_ODATA_SERVICES_V2SERVICE_AUTH_USERNAME=admin
```

Das Muster ist `NUXT_ODATA_SERVICES_<NAME>_<PROPERTY>`, wobei `<NAME>` der in Großbuchstaben geschriebene Service-Name ist.
