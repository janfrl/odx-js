Das `useOData` Composable wird automatisch vom Modul `@me-tools/odx-nuxt` importiert. Es bietet einen typsicheren, reaktiven Wrapper um Nuxts `useFetch`.

## Nutzungsmuster

### Punkt-Notation (Empfohlen)

Bietet vollständige IDE-Autovervollständigung für Service- und Entitätsnamen basierend auf Ihren generierten Typen.

```ts
const { data, pending, error } = await useOData().V2Service.Products.list({
  $select: ['ProductID', 'ProductName'],
  $filter: 'UnitPrice gt 20'
})
```

### Funktionale Syntax

Verwenden Sie dies für dynamische Service- oder Entitätsnamen (z. B. aus Routen-Parametern).

```ts
const service = 'V2Service'
const entity = 'Products'

const { data } = await useOData(service).entitySet(entity).list()
```

## Methoden-Referenz

| Methode | HTTP | Rückgabewert | Beschreibung |
| --- | --- | --- | --- |
| `.list(query?, options?)` | `GET` | `AsyncData<T[]>` | Ruft eine Sammlung von Entitäten ab. |
| `.get(id, query?, options?)` | `GET` | `AsyncData<T>` | Ruft eine einzelne Entität über die ID ab. |
| `.create(payload)` | `POST` | `Promise<T>` | Erstellt eine neue Entität. |
| `.update(id, payload)` | `PATCH` | `Promise<T>` | Aktualisiert eine bestehende Entität. |
| `.remove(id)` | `DELETE` | `Promise<void>` | Entfernt eine Entität. |
