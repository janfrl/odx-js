Der ODX Explorer ist Ihr Hauptwerkzeug zum Debuggen von OData-Services
waehrend der Entwicklung. In einer lokalen Nuxt App ist er direkt in die
**Nuxt DevTools** eingebettet und erfordert keine zusaetzliche Installation,
wenn `@bc8-odx/nuxt` verwendet wird.

::note
Der Explorer ist automatisch verfügbar, wenn `@bc8-odx/nuxt` in `modules` registriert ist und `devtools: { enabled: true }` im `odata`-Block gesetzt ist (Standard).
::

::warning
Der lokale DevTools Explorer unterscheidet sich vom eigenstaendigen Explorer,
der hinter dem SAP BTP AppRouter deployed wird. Lokale DevTools koennen
entwicklungsbezogene Konfiguration, generierte Typen, rohe Metadaten und
In-Memory-Traffic-Logs anzeigen. Produktions-APIs unter `/__odx__` benoetigen
SAP Security Context und geben nur reduzierte, bereinigte Daten zurueck.
::

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  odata: {
    devtools: {
      enabled: true,
      maxLogs: 100
    }
  }
})
```

:::steps{level="2"}

## Nuxt DevTools öffnen

Starten Sie Ihre Nuxt-Anwendung im Entwicklungsmodus:

```bash
pnpm run dev
```

Öffnen Sie Ihren Browser und drücken Sie `Umschalt + Alt + D` oder klicken Sie auf das Nuxt-Logo am unteren Bildschirmrand.

## Den ODX-Tab finden

Suchen Sie in der DevTools-Seitenleiste nach dem **ODX**-Icon. Ein Klick darauf öffnet die Explorer-Oberfläche.

## Schemata erkunden

Wählen Sie einen Ihrer konfigurierten Services aus dem Dropdown-Menü aus. Sie sehen eine Liste der verfügbaren Entitäts-Sets. Ein Klick auf ein Entitäts-Set zeigt dessen Eigenschaften, Schlüsselfelder und Navigationsbeziehungen an.

## Anfragen überwachen

Lösen Sie eine OData-Anfrage in Ihrer Anwendung aus (z. B. indem Sie zu einer Seite navigieren, die `useOData` verwendet). Wechseln Sie zum Tab **Traffic** im Explorer, um die Request-URL, Header, Payload und die Backend-Antwort zu sehen.

Traffic-Logs sind Entwicklerdiagnosen und koennen sensible Daten enthalten.
Redigieren Sie ausgehende Header, Auth-/Session-/CSRF-Werte, Request-Bodies und
Response-Bodies, bevor Sie Logs exportieren oder kopieren. Grosse Payloads
sollten auch in der lokalen Entwicklung gekuerzt oder begrenzt werden.

## Mock-Daten exportieren

Im Tab **Data Browser** können Sie Entitätsdaten als JSON herunterladen. Platzieren Sie die exportierten Dateien in `.odx/mock/`, um sie für die Offline-Entwicklung zu nutzen.
