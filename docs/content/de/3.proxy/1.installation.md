Verwenden Sie den eigenständigen Proxy für ein Backend-for-Frontend (BFF) Muster, eine Microservice-Architektur oder jedes andere Framework außer Nuxt.

:::steps{level="3"}
### Abhängigkeiten installieren

```bash
pnpm add @bc8-odx/proxy h3
```

### Handler erstellen

Verwenden Sie das Utility `createODataHandler`, um einen Standard H3 Event-Handler zu erstellen.

```ts [server.ts]
import { createODataHandler } from '@bc8-odx/proxy'

const config = {
  services: [
    { name: 'Northwind', url: 'https://services.odata.org/V2/Northwind/Northwind.svc/' }
  ]
}

export const odataHandler = createODataHandler(config)
```

### Route einbinden

Der Handler ist kompatibel mit jedem Standard H3 oder Nitro Server.

```ts [nitro.config.ts]
export default defineNitroConfig({
  modules: ['@bc8-odx/proxy/nitro'],
  odata: {
    services: [
      { name: 'MyService', url: '...', destination: 'MY_DESTINATION' }
    ]
  }
})
```
:::

## Konfigurationsoptionen

Das Proxy-Konfigurationsobjekt spiegelt den `odata`-Block des Nuxt-Moduls wider. Alle [Service-Definitionsoptionen](/de/nuxt/module-reference#odataservices) des Nuxt-Moduls gelten auch hier.
