Dieser Leitfaden bietet Lösungen für häufige Probleme, auf die Sie bei der Verwendung von ODX stoßen könnten.

:::accordion

:::accordion-item{label="1. Typgenerierung schlägt fehl" icon="i-lucide-file-warning"}
Wenn `npx nuxi prepare` keine Typen für Ihre Services generiert:

- **URL prüfen**: Stellen Sie sicher, dass die `url` in `nuxt.config.ts` korrekt und erreichbar ist.
- **EDMX-Erreichbarkeit**: Wenn die URL auf einen Live-Service verweist, muss ODX in der Lage sein, die `$metadata` während des `prepare`-Schritts ohne Authentifizierung abzurufen (oder Sie müssen Anmeldedaten angeben).
- **Lokale EDMX**: Versuchen Sie, die EDMX-Datei herunterzuladen und die `url` auf den lokalen Pfad zu verweisen.
:::

:::accordion-item{label="2. Authentifizierungsfehler (401/403)" icon="i-lucide-lock"}
- **Proxy-Strategie**: Stellen Sie sicher, dass Sie `strategy: 'proxied'` verwenden, wenn ODX die Basic/Bearer/BTP-Authentifizierung übernehmen soll.
- **BTP Destinationen**: Überprüfen Sie, ob der Name der Destination exakt mit dem im BTP Cockpit übereinstimmt.
- **Principal Propagation**: Stellen Sie sicher, dass der Benutzer über XSUAA an der Nuxt App angemeldet ist.
:::

:::accordion-item{label="3. CSRF-Token-Probleme" icon="i-lucide-shield-check"}
- **HEAD-Anfragen**: Einige SAP-Systeme erfordern `GET` anstelle von `HEAD` für CSRF. Überprüfen Sie Ihre Backend-Konfiguration.
- **Proxy-Logs**: Aktivieren Sie die DevTools-Logs, um zu sehen, ob das CSRF-Pre-Fetching erfolgreich ist.
:::

:::accordion-item{label="4. Explorer wird nicht angezeigt" icon="i-lucide-eye-off"}
- **Dev-Modus**: Der Nuxt DevTools Explorer funktioniert nur im Entwicklungsmodus (`nuxt dev`).
- **Modul-Reihenfolge**: Stellen Sie sicher, dass `@bc8-odx/nuxt` in `modules` registriert ist.
- **DevTools aktiviert**: Überprüfen Sie, ob `devtools: { enabled: true }` im `odata`-Block gesetzt ist.
- **Produktions-Explorer**: In BTP nutzen Sie den eigenstaendigen Explorer
  hinter dem AppRouter. `/__odx__/client/*` sollte zur Explorer UI routen,
  waehrend nur `/__odx__/{config,logs,schema,generate,types,me}` zum Proxy
  routen sollte.
:::

:::accordion-item{label="5. Produktions-Explorer-Logs sind leer" icon="i-lucide-list-x"}
Produktions-`/__odx__/logs` gibt eine leere Liste zurueck und lehnt Loeschen
mit `403` ab, solange SQL-Log-Speicherung nicht explizit konfiguriert ist.
Aktivieren Sie den proxy-eigenen db0-backed Store mit
`NUXT_ODATA_DEVTOOLS_LOG_STORE=sql` und einer verwalteten SQL-Datenbank, wenn
deployte Traffic-Historie benoetigt wird. Gespeicherte Produktionseintraege
bleiben redigiert, Request- und Response-Payload-Bodies werden standardmaessig
weggelassen, und der Explorer verbindet sich nie direkt mit der Datenbank.
:::

:::accordion-item{label="6. Proxy gibt 502 Bad Gateway zurück" icon="i-lucide-server-off"}
- **Backend-URL**: Bestätigen Sie, dass die Backend-URL vom Server aus erreichbar ist, auf dem Nitro läuft.
- **On-Premise-Systeme**: Wenn das Backend On-Premise ist, stellen Sie sicher, dass der BTP Connectivity Service gebunden ist und die Destination mit `ProxyType: OnPremise` konfiguriert ist.
:::
