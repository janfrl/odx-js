Beiträge sind willkommen! Dieser Leitfaden erklärt, wie man das Monorepo lokal zum Laufen bringt und Änderungen einreicht.

## Monorepo-Struktur

ODX ist ein **pnpm workspace** Monorepo. Die oberste Struktur ist:

```
nuxt-sap-odata/
├── packages/
│   ├── core/          # @me-tools/odx-core
│   ├── nuxt/          # @me-tools/odx-nuxt
│   ├── proxy/         # @me-tools/odx-proxy
│   └── explorer/      # @me-tools/odx-explorer
├── docs/              # Diese Dokumentationsseite
├── specs/             # Feature-Spezifikationen und Planungsdokumente
└── pnpm-workspace.yaml
```

## Voraussetzungen

- **Node.js** >= 20
- **pnpm** >= 9

## Einrichtung

```bash
# Repository klonen
git clone https://github.com/me-tools/odx.git
cd odx-js

# Alle Abhängigkeiten im gesamten Workspace installieren
pnpm install

# Alle Pakete bauen (erforderlich vor dem Ausführen der Dokumentation)
pnpm build
```

## Ausführen der Dokumentationsseite

```bash
cd docs
pnpm dev
```

Die Dokumentationsseite läuft unter [http://localhost:3000](http://localhost:3000).

## Tests ausführen

```bash
# Tests für alle Pakete ausführen
pnpm test

# Tests für ein einzelnes Paket ausführen
pnpm --filter @me-tools/odx-core test
```

## Änderungen einreichen

1. Forken Sie das Repository und erstellen Sie einen Feature-Branch von `main`.
2. Nehmen Sie Ihre Änderungen mit klaren, fokussierten Commits vor.
3. Stellen Sie sicher, dass `pnpm build` und `pnpm test` erfolgreich durchlaufen.
4. Öffnen Sie einen Pull Request mit einer Beschreibung, was geändert wurde und warum.

Für reine Dokumentationsänderungen reicht ein Build der Dokumentationsseite aus.
